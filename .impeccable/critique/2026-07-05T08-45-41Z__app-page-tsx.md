# Critique — hivePOS Public Landing (brand surface)

## Method
`Method: dual-agent (A: design-director review sub-agent · B: deterministic detector + best-effort browser)`. Assessment A ran isolated from detector output. Assessment B = `detect.mjs` CLI scan + Playwright browser pass (dev server up on :3007; the Playwright session was authenticated so `/` and `/register` redirected to `/dashboard` — public routes weren't visually captured; source + detector carry the findings).

## Anti-Patterns Verdict
**LLM assessment:** *The parts are tasteful, the whole is generic.* A designer looking cold would call it "a clean, competent, sky-blue SaaS landing — probably AI-assisted." Tells: (1) the hero "proof band" is the textbook hero-metric template (`LandingHero.tsx:67-79`) — 4-up big-number + small-label, with two cells that are dates ("Okt 2024", "Jun 2026") masquerading as stats to fill the grid (the giveaway); (2) 14-section vertical cadence is the stamped SaaS outline, no rest beat; (3) four near-identical "centered headline + max-w-2xl lede" blocks (FeatureBento/HowItWorks/PricingSection/FAQ) — the AI scaffold cadence; (4) metronomic white↔sky-50 background oscillator; (5) two fullscreen blue shouts back-to-back (BetaPartner sky-600, FinalCTA sky-700) — the closer loses punch. **Rescued by:** `pos-preview.tsx` renders *real* order markup (tabular nums, real `HBL-0007` codes, Rp figures) — anti-slop; honest FAQ copy (iPhone-printing limit, manual entry); PricingSection is one bordered container with shared hairlines, not 3 floating towers.
**Deterministic scan:** 1 finding — `gray-on-color: text-slate-700 on bg-sky-50` at `LandingHero.tsx:44` (warning). Real; the slate-700 on sky tint reads washed-out. No false positives.
**Browser:** public routes redirect when authenticated; not visually captured. Source-confirmed.

## Cognitive Load
Failures: >4 options at decision points (FinalCTA 3 same-weight pills; hero 2 same-weight CTAs; **8 CTA labels** across page); page length (14 blocks, no TL;DR); WebsiteSpotlight dumps **14 sub-features** across 4 groups (feature-dump-inside-a-feature — fatigue peak); Pricing 3 tiers with overlapping lists (no diff toggle). Pricing 3 tiers is fine; FAQ 9 borderline-acceptable but ungrouped. Anchor nav needs `scroll-mt` offset for the sticky 64px nav (verify).

## Emotional Journey
**Peak (hero):** "Kasir laundry, tinggal buka browser." with sky emphasis on the hook is the brand promise made literal — but immediately diluted by the proof band appearing *before the user reads a single benefit*. **Trust-build:** `pos-preview` ("Honey Bee Laundry / 18 order hari ini / Omzet Rp 792.000") + "Tampilan asli dari laundry kami sendiri" caption is the strongest trust element — dogfooding correctly positioned as supporting proof, not hero. **Friction at signup:** the register page breaks the spell — a `bg-gradient-to-br from-indigo-500 to-indigo-700` brand tile (`register/page.tsx:107-109`) on a `.pub-scope` page, clashing with the sky landing the user just left (Two-Hue violation at the conversion door). **End:** BetaPartner + FinalCTA = two blue shouts back-to-back → blue fatigue → closer lands flat. **Verdict:** promise is *lightness*; the page is *heavy* — central emotional dissonance.

## Strengths
1. **`pos-preview.tsx` is the page's best asset** — real order rows, tabular nums, hairline dividers, single brand accent reserved for the action-needed chip. Product-as-marketing done right.
2. **Copy honesty above the bar** — FAQ #7 (iPhone Bluetooth/USB print unsupported in Safari), FAQ #5 (manual entry if no export), pricing footnote ("Harga per outlet, bukan per user"). The "jujur" pillar made literal.
3. **PricingSection = one bordered container with shared hairlines**, highlight tier by tint + corner tag, not elevation/scale — resists the 3-tower SaaS reflex.
